What is it?

sfdoc is a Django web app that runs on Heroku and publishes documentation from an easyDITA instance to a Salesforce org as Knowledge Articles.

How does it work?

HTML output is generated by easyDITA using the DITA Open Toolkit and sent to sfdoc where it is scrubbed using a set of whitelists. Images are uploaded to Amazon S3 and image links in the HTML are updated accordingly. Two Salesforce orgs are needed: One is the production org and the other is a sandbox org to use for reviewing changes before going live in production.

Can I use it with my own easyDITA instance and Salesforce orgs?

Yes! sfdoc is an open source project developed by Salesforce.org and hosted at GitHub. For information on setting up your own instance of sfdoc, please see the repo readme file.
